## Agent Automation Approaches - CAPTURED

**Problem:** How to complete tasks when APIs don't exist (e.g., restaurant reservations)

**Options ranked by reliability:**

1. **APIs** (most reliable) — Direct integration where available
2. **MCP Servers** (emerging) — Model Context Protocol, agent-native endpoints
3. **Middleware** (Zapier/Make/N8N) — They handle scraping/APIs, I call their API
4. **Desktop Automation** (RPA) — Playwright/Puppeteer on local machine
5. **Browser Automation** (fragile) — Headless Chrome on VPS, high maintenance
6. **Human-in-the-Loop** (expensive) — Draft requests, human sends/executes

**Key insight:** Browser automation on VPS is possible but resource-heavy and brittle. Better to use APIs → middleware → human delegation for production tasks.
