# Session: April 7, 2026 — Social Credibility System Design

## Project: Non-repudiable Social Posts on Cardano

### Overview
Designed a meritocratic credibility system using Cardano's native asset infrastructure. Core principles: non-repudiation, immutability, UX-optimized.

### Tech Stack Decisions
- **Token Standard**: CIP-25 (immutable NFT) — no edits, true immutability
- **Mutable Pointers**: CIP-68 for head registry (optional)
- **Storage**: IPFS for content, hashes on-chain
- **Recovery Contract**: Aiken validator with on-chain verification
- **Wallet**: Mesh.js or Lucid

### Hash Chain Structure
Each post links to previous via `prev_post_hash`:
```
Genesis → Post_1 (seq: 1) → Post_2 (seq: 2) → ... → Post_N
```

Properties:
- Sequence integrity (missing number = deleted post)
- Tamper detection (broken hash chain)
- Fork detection (two posts claiming same prev)

### Optimistic Recovery Mechanism
Problem: User loses head hash → can't append

Solution: Cryptoeconomic checkpoint recovery
- Recovery bond: 50 ADA
- Challenge window: 7 days
- Challenge bond: 5 ADA
- Successful challenge: 80% to challenger, 20% burned

### On-Chain Challenge Verification
Requires merkle proof (no off-chain indexers):
- Challenger provides: `later_post`, `merkle_path`, `merkle_root`, `mint_reference`
- Contract verifies: path proves ancestry + CIP-25 mint is valid

Merkle tree:
- Leaves: `hash(0x00 || content_hash)`
- Branches: `hash(0x01 || left || right)`

### UX Optimizations
- Session signing (1 hour, batch posts)
- Gasless submission (platform pays, user signs intent)
- Batch minting (5-10 posts/tx, ~0.05 ADA/post)
- Draft → commit flow (local storage until publish)
